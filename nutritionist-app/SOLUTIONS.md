# 🍎 營養師 App 圖片識別解決方案

**問題：** 阿里雲 Coding Plan (¥39.9/月) 唔支援圖片上傳

**解決：** 用免費/開源方案做圖片識別 + Coding Plan 做營養分析

---

## 🎯 三個可行方案

### 方案 A：Hugging Face 免費 API（最簡單）

**原理：**
```
圖片 → Hugging Face API (免費) → 食物名稱 → Coding Plan → 營養分析
```

**優點：**
- ✅ 無需安裝任何嘢
- ✅ 5 分鐘搞掂
- ✅ 完全免費（1000 次/月）

**缺點：**
- ⚠️ 有速率限制（每分鐘約 10 次）
- ⚠️ 需要網絡連接

**成本：**
- 圖片識別：免費
- 營養分析：包喺 ¥39.9/月 內
- **總計：¥39.9/月（無額外費用！）**

**運行方法：**
```bash
cd /home/node/.openclaw/workspace/nutritionist-app
python3 food_recognition_hybrid.py <圖片路徑>
```

---

### 方案 B：本地 BLIP 模型（最穩定）

**原理：**
```
圖片 → 本地 BLIP 模型 → 食物描述 → Coding Plan → 營養分析
```

**安裝：**
```bash
pip3 install transformers pillow torch
```

**優點：**
- ✅ 無限次使用
- ✅ 無需網絡
- ✅ 私隱性好（圖片唔使上傳）

**缺點：**
- ⚠️ 首次安裝需下載模型（約 2GB）
- ⚠️ 需要一定計算資源

**成本：**
- 圖片識別：免費（本地運行）
- 營養分析：包喺 ¥39.9/月 內
- **總計：¥39.9/月（無額外費用！）**

---

### 方案 C：Google Vision API（最準確）

**原理：**
```
圖片 → Google Vision API → 食物標籤 → Coding Plan → 營養分析
```

**免費額度：** 1000 次/月

**優點：**
- ✅ 識別最準確
- ✅ 支援多種食物
- ✅ 穩定可靠

**缺點：**
- ⚠️ 需要信用卡註冊
- ⚠️ 可能需翻牆
- ⚠️ 超過免費額度要付費

**成本：**
- 圖片識別：免費（1000 次/月內）
- 營養分析：包喺 ¥39.9/月 內
- **總計：¥39.9/月（無額外費用！）**
